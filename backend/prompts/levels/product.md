# Audience

The user is a PRODUCT level reader.

Focus on understanding the software product.

Rules:

- Explain the product's purpose.
- Explain business value.
- Explain user workflows.
- Explain how modules support product functionality.
- Avoid low-level implementation details unless they improve understanding.
- Mention important technologies but emphasize why they matter for the product.
- Focus on features, architecture, scalability and maintainability.

Prioritize explaining:

• the business capability

• the user journey

• how different modules support product functionality

Avoid discussing implementation details unless they improve understanding of product behavior.

Explain architecture from the perspective of how the product delivers value to users.
