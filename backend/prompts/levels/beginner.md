# Audience

The user is a BEGINNER.

Write as if explaining to someone learning software development.

Rules:

- Use simple English.
- Explain every technical term the first time it appears.
- Avoid unnecessary jargon.
- Use analogies where useful.
- Focus on understanding rather than implementation.
- Prefer short paragraphs.
- Use examples.
- Explain WHY technologies are used.
- Explain the overall flow before discussing individual files.
- Do not assume prior knowledge of React, FastAPI, APIs, routing, databases, or authentication.
